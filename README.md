
STM32F407 四轮舵轮底盘控制系统

本项目是基于大疆开发板 C 型（MCU: STM32F407IGT6）开发的四轮舵轮（SwerveDrive）底盘控制系统。系统实现了遥控器信号解析、底盘运动学统一解算、舵机零偏校准、舵向电机就近转向算法以及双环 PID 闭环控制。底盘控制核心逻辑运行在
FreeRTOS 任务中。

开发环境与硬件依赖

软件环境

  - IDE / 工具链：CLion (CMake) + arm-none-eabi-gcc
  - 底层配置：STM32CubeMX (HAL 库)
  - 调试与烧录：DepLink / Ozone + OpenOCD / VOFA+ 上位机

硬件架构与 CAN 总线分配

由于单路 CAN 总线带宽限制，为避免在 1KHz 反馈频率下产生总线拥堵（导致标识符较大的 0x208 舵电机数据丢包），系统采用双路 CAN 总线分流方案：

  - CAN2：挂载 4 个 M3508 驱动轮电机（反馈 ID: 0x201 ~ 0x204）
  - CAN1：挂载 4 个 GM6020 舵向轮电机（反馈 ID: 0x205 ~ 0x208）
  - 遥控器：大疆接收机（挂载于 USART3，采用 DBUS 协议解析）

2. 舵轮就近转向逻辑

为防止舵电机在目标角度与当前角度偏差大于 90° 时进行长距离旋转，解算中引入了就近转向算法：

  - 当目标角度与当前角度的偏差绝对值大于 90°（\frac{\pi}{2} rad）时，将目标角度偏转 \pm180°，同时将 M3508
    驱动轮电机的控制速度取反。
  - 该机制有效缩短了舵向电机的调整路径，提升了底盘的动态响应速度。

调试与校准指南

1. 舵轮零偏校准

新车装配或拆卸舵机后，必须重新校准 GM6020 电机的物理零偏，否则会导致解算角度偏差。

校准步骤：

1.  将车体架高（使四个轮子悬空），手动将四个舵轮物理方向调整至绝对朝前（与车头方向平行）。
2.  连接 DepLink 仿真器，在 Debug 状态下在线读取四个 GM6020 电机当前的反馈转子角度值（ecd，范围 0 ~ 8191）。
3.  将读取到的数值依次写入 Chassis_Task.c 中的 GM6020_OFFSET 数组中进行物理校准：

// 对应 1 号 ~ 4 号舵电机的零偏机械角度反馈值
const int16_t GM6020_OFFSET[5] = {0, 1350, 1350, 5446, 5446};

2. PID 参数整定

  - GM6020 舵向电机：采用“外环角度 + 内环速度”的双环 PID 进行位置跟踪。
      - 整定顺序：优先整定内环速度环 Kp 与 Ki，确保速度响应无明显震荡；随后引入外环角度 Kp，使角度跟踪紧凑无超调。
  - M3508 驱动电机：采用单速度闭环控制。
      - 注意：若驱动轮 PID 差值过大，底盘在平移时易出现走歪或直线跑偏现象，需保证四轮控制参数的一致性。

编译与烧录

1.  使用 CLion 打开本项目。
2.  检查 CMake 工具链中的编译器路径是否正确指向 arm-none-eabi-gcc。
3.  点击 CLion 顶部的 Build 按钮进行编译。
4.  确认生成 .elf 或 .hex 文件后，使用 DepLink 或其他调试器烧录到主控板，复位运行。
